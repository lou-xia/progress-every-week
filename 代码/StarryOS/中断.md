## 中断和调度如何解耦

在 StarryOS 的`api/src/mm.rs`的`handle_page_fault()`有注解`#[register_trap_handler(PAGE_FAULT)]`；在`arceos/modules/axhal/src/irq.rs`的`irq_handler()`有注解`#[register_trap_handler(IRQ)]`。对这二者进行分析

### DistributedSlice 类

在`linkme`库中定义了`DistributedSlice`类，同时对外暴露了宏`distributed_slice`。

#### 作用

可以收集不同 crate 定义的对象，并合成一个静态对象数组。

#### 使用方法

定义对象数组：通过在要定义的对象数组上加入注解`#[distributed_slice]`，定义一个静态对象数组。

注册对象：通过在要注册的对象上加入注解`#[distributed_slice(Name)]`，括号内的`Name`表示要注册到哪个对象数组上。

例如，在 StarryOS 的`axcpu`模块中`axcpu/src/trap.rs`，定义 irq 处理函数的数组（在 use 时改了个名字）：

```rust
pub use linkme::distributed_slice as def_trap_handler;
pub use linkme::distributed_slice as register_trap_handler;

/// A slice of IRQ handler functions.
#[def_trap_handler]
pub static IRQ: [fn(usize) -> bool];
```

然后在`arceos/modules/axhal/src/irq.rs`的`irq_handler()`上定义：

```rust
#[register_trap_handler(IRQ)]
pub fn irq_handler(vector: usize) -> bool {
    ... // 具体内容
    true
}
```

使用`irq_handler`，位置是`arceos/modules/axhal/src/irq.rs`：

```rust
{
    let mut iter = crate::trap::IRQ.iter();
    if let Some(func) = iter.next() {
        if iter.next().is_some() {
            warn!(
                "Multiple handlers for trap {} are not currently supported",
                stringify!(IRQ)
            );
        }
        func(scause.bits())
    } else {
        warn!(
            "No registered handler for trap {}",
            stringify!(IRQ)
        );
        false
    }

}
```

这时`IRQ`中已经有处理函数`irq_handler`了，即`pub static IRQ: [fn(usize) -> bool] = [irq_handler];`

#### 原理

在定义数组时，注解会把原定义展开，把类型`[T]`改写成`DistributedSlice<[T]>`，然后在二进制文件中创建两个 linker symbol：start 和 stop，运行时用这两个地址构造数组。此外还有生成额外的 section 以及重复检查机制。

最终变成类似这样的内容：
```rust
pub static IRQ: linkme::DistributedSlice<[fn(usize) -> bool]> = {
    extern "Rust" {
        static __start_linkme_IRQ: fn(usize) -> bool;
        static __stop_linkme_IRQ: fn(usize) -> bool;
    }

    #[used]
    #[link_section = "linkme_IRQ"]
    static mut LINKME_PLEASE: [fn(usize) -> bool; 0] = [];

    unsafe {
        linkme::DistributedSlice::private_new(
            "IRQ",
            &__start_linkme_IRQ,
            &__stop_linkme_IRQ,
        )
    }
};
```

在编译中间文件.o中会有`linkme_IRQ`和`linkm2_IRQ`。这个过程不需要手动修改链接文件。

在注册对象时，注解会展开，把注册的对象链接到`linkme_IRQ`节中，变成类似这样：

```rust
#[used]
#[link_section = "linkme_IRQ"]
static IRQ_HANDLER: fn(usize) -> bool = irq_handler;
```

从而变成一个数组。

此外，`DistributedSlice`实现了迭代器的 trait，因此才可以直接调用`.next()`
