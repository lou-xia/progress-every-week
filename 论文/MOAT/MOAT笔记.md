## 我的理解
这个文章提出了一个模型（或者说是方案？），实现逐步改进现有的商用的内核，提高安全性，支持热更新。

架构：
1. Trusted Computing Base (TCB)：包括：
   1. 域库：封装所有 Rust 编写的 Unsafe 代码，用于对内核底层资源的控制，比如内存访问和硬件访问，对外提供安全的接口。其中还包括了**私有堆、共享堆和自定义堆**。
   2. 运行时：管理域的加载、更新和页面等资源的分配，
2. 隔离域们：完全安全的 Rust 编写，实现动态加载和热更新。  
3. 其他的没修过的内核模块。

借由 Rust 语言的性质，实现将不安全的代码限制在 TCB 中，TCB 外的代码都是安全的。

隔离域之间，借助上面的三个堆实现隔离与通信：
- 私有堆：每个域内的堆，只能被该域访问。
- 共享堆：所有域共享一个堆，利用DBox<>实现
- 自定义堆：热更新时保存状态。

出错隔离：借助 Rust 的 Panic Hook 实现（这部分不太懂）

热更新：
- 无状态的域：被当作数据对象，使用 RCU 机制更新
- 有状态的域：通过加锁、计数器、IPI （~~不知道是啥，之后查查看~~）实现。
